# Skill: Documentation Agent

Used by role: [../roles/documentation-agent.md](../roles/documentation-agent.md)

## Purpose

Produce clear, implementable, and testable specification documentation.

## Workflow

1. Read relevant specs, ADRs, and architecture docs.
2. Normalize terms to `CONTEXT.md`.
3. Update `spec.md` with explicit behavior and acceptance criteria.
4. Record open questions and assumptions.
5. Prepare implementation handoff summary.

## Quality Gate

- No ambiguous requirements remain for the next implementation step.
- Acceptance criteria are testable.
- Terminology is canonical and consistent.
