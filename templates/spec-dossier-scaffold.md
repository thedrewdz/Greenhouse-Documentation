# Spec Dossier Scaffold

Use this scaffold when creating a new spec dossier under `specs/<spec-name>/`.

## Required

- `spec.md`

## Recommended Lifecycle Files

- `implementation-plan.md`
- `review-report.md`
- `qa-report.md`
- `retrospective.md`
- `doc-feedback.md`

## Quick Start

1. Create a folder at `specs/<spec-name>/`.
2. Copy templates from this folder into the new dossier.
3. Start by filling `spec.md` first.
4. Add lifecycle files as each stage completes.

## Suggested Naming

- Use kebab-case folder names.
- Keep one feature per dossier folder.
- Keep contract-only specs in their own dedicated dossier folder.
